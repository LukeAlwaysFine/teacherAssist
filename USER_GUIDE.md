## 快速开始

### 1. 注册/登录

打开应用 → 注册一个新账号（邮箱 + 密码 + 姓名）→ 登录。

登录状态保持 **30 天**，关闭浏览器后再次打开无需重新登录。

### 2. 配置 AI

首次使用必须先配置 LLM API。详见 [配置 AI 大模型](#配置-ai-大模型)。

### 3. 三步流程

页面顶部有 **3 个步骤**：

```
● 1 课堂信息  ▸  ○ 2 录制转录  ▸  ○ 3 分析报告
```

逐步完成每一步。未完成前一步时，后续步骤会显示 🔒 锁定状态，点击会弹出提示。

## 配置 AI 大模型

### 为什么需要配置？

系统不预设任何 AI 服务。你需要提供自己的 API Key 才能使用课堂分析和报告生成功能。

### 操作步骤

1. 点击页面右上角的 **⚙️ API** 按钮
2. 在弹出的设置窗口中填入：

| 字段 | 怎么填 |
|------|--------|
| **提供商名称** | 填写你使用的服务名，如 `deepseek`、`openai`、`anthropic`、`groq`、`ollama` |
| **API Key** | 你的 API 密钥。从服务商后台获取，通常以 `sk-` 开头 |
| **模型 ID** | 要使用的模型名称，如 `deepseek-chat`、`gpt-4o`、`claude-sonnet-4-6` |
| **最大 Token 数** | 每次生成的最大字数限制，默认 4096 即可 |
| **API 地址** | OpenAI 兼容服务的端点 URL，如 `https://api.deepseek.com`。注：Anthropic 留空 |

3. 点击 **💾 保存配置，启用 AI 功能**

### 各服务商配置示例

| 服务 | 提供商名称 | API 地址 | 模型 ID 示例 |
|------|-----------|---------|-------------|
| DeepSeek | `deepseek` | `https://api.deepseek.com` | `deepseek-chat` |
| OpenAI | `openai` | `https://api.openai.com/v1` | `gpt-4o` |
| Anthropic | `anthropic` | 留空 | `claude-sonnet-4-6` |
| Groq | `groq` | `https://api.groq.com/openai/v1` | `llama-3.1-70b-versatile` |
| 本地 Ollama | `ollama` | `http://localhost:11434/v1` | `qwen2:7b` |

### 说明

- Anthropic 填写 `anthropic` 时使用官方 SDK，其他名称均走 OpenAI 兼容协议
- 配置保存在你的账号下，不会与其他用户共享
- 修改配置后**立即生效**，无需重启

## 创建课堂

### Step 1：课堂信息

1. 选择科目、教材、年级
2. 在大纲树中勾选知识点（或点「全选」）
3. 已选知识点自动填充到下方面板
4. 填写课堂标题、学生姓名、上课时间
5. 点击 **创建课堂 → 进入下一步**

> 提示：已选知识点面板默认折叠，有选择后自动展开并显示条数。

## 获取转录文本

### Step 2：录制转录

页面提供 **三种方式** 获取转录文本，所有方式的结果显示在同一个转录框中：

#### 方式一：🎤 实时录制

1. 选择切片时长（推荐 5 分钟）
2. 点击 **开始录制**，使用浏览器麦克风
3. 实时转录文字持续更新
4. 录制完成后自动转录

#### 方式二：📁 上传音频文件

1. 选择音频文件（支持 mp3/wav/m4a/webm 等）
2. 选择转录引擎
3. 点击 **上传并开始转录**
4. 等待进度条完成

#### 方式三：📄 复用已有文档

1. 从历史课堂列表中选择已转录的文档
2. 确认后直接导入当前课堂

### 转录完成后

- 转录文字显示在 **📝 转录文字** 统一框中（带来源标签和字数统计）
- 底部出现 **转录完成 → 进入分析报告** 按钮（蓝色可点击）
- Step 3 自动解锁，1.5 秒后自动跳转

## AI 分析课堂

### Step 3：分析报告

#### 可选：输入教师定性观察

在知识点大纲预览下方，有一个 **💬 教师定性观察** 折叠面板。你可以在此输入对学生的观察，例如：

- "注意力集中，反应快"
- "今天状态不太好，容易分心"
- "对亚洲地理分区特别感兴趣"

AI 会结合你的观察生成更准确的家长报告。留空则完全基于转录数据分析。

#### 触发分析

1. 确认知识点大纲无误（可在折叠面板中编辑）
2. 点击 **🤖 开始 AI 分析**
3. 等待进度条完成（通常 30-90 秒）
4. 分析完成后自动展示：
   - 清理后的转录文本
   - 知识点覆盖情况（含掌握程度和证据）
   - 课堂互动统计
   - 巩固建议

#### 报告内容

- 🟢 已掌握 / 🟡 存疑 / 🔴 未掌握 / ⚪ 待确认 / 灰色—未覆盖
- 每个知识点可查看「证据」——转录中的具体对话
- 参与度、提问次数、师生对话比

## 家长反馈报告

### 自动生成

分析完成后，系统自动基于分析结果生成家长报告。报告包含三段结构：

- **【授课内容】** — 课堂实际讲的知识点
- **【上课情况】** — 学生掌握程度 + 课堂状态
- **【课后任务】** — 具体可执行的巩固练习

### 报告操作

| 操作 | 说明 |
|------|------|
| 💾 保存修改 | 直接编辑报告文字后保存 |
| 🤖 重新生成 | 输入修改建议让 AI 修订，或留空全文重生成 |
| 📷 生成图片 | 一键导出 PNG 图片，方便微信/钉钉发送给家长 |

### 切换模板

报告区域顶部有 **报告模板** 下拉框：
- 选择「学达默认模板」使用系统自带格式
- 选择你自己的模板使用自定义格式
- 点击 📂 管理模板上传和管理模板

## 报告模板管理

### 上传模板

1. 点击报告区域的 **📂 管理模板**
2. 填写模板名称
3. 选择 `.txt` 模板文件
4. 点击上传

### 模板变量

模板中可使用以下占位符，生成时自动替换：

```
{subject}              学科
{date}                 日期
{time}                 时间
{student_name}         学生姓名
{total_knowledge_points}  知识点总数
{covered_count}           已覆盖数
{mastered_count}          已掌握数
{engagement_level}        参与度
```

分析数据（知识点列表、互动统计、巩固建议）以 JSON 格式传入，LLM 自动理解。

### 设为默认

在模板列表中点击 ⭐ **设为默认**，之后生成报告自动使用该模板。

## 常见问题

### Q: 点击分析按钮提示"未配置 LLM API Key"

**A:** 点击右上角 **⚙️ API**，填入你的 API Key 后重试。

### Q: 报告中出现"掌握情况尚不明确"等模糊表述

**A:** 系统已禁止此类表述。如果数据不足以评价某个知识点，报告会跳过不提，而不是写"尚不明确"。如果仍然出现，可以点击「🤖 重新生成」让 AI 修订。

### Q: 转录文字不显示

**A:** 按 `Ctrl+F5` 强制刷新页面。确保转录已完成（状态栏显示"转录完成"）。

### Q: 大纲中没有我需要的科目

**A:** 目前支持：人教版数学/历史/地理（初高中）、浙教版初中科学。如需其他教材请联系开发者。

### Q: 可以多个老师共用吗

**A:** 可以。每个老师注册独立账号，数据完全隔离。LLM 配置也各自独立。

### Q: 报告可以修改后重新生成吗

**A:** 可以。在修改建议输入框中描述你想改的内容，AI 会在保留原有正确内容的基础上只修改你提到的部分。
